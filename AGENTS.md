## Контекст проекта

**INCF Intro to Neurocoaching** — геймифицированное асинхронное мини-приложение курса.
Студенты смотрят короткие YouTube-видео, выполняют микро-задания, зарабатывают XP и
значки, проходят финальный тест и попадают в воронку продаж (диагностический звонок → L1).

Платформа живёт на `incf-neurocoaching.vercel.app`, репозиторий: `pine-praire/incf-neurocoaching`.

## Дизайн-система — обязательно к соблюдению

Токены живут в `styles/tokens.css`. **Никогда не вписывай hex в компонент.**

- **Цвета:** `bg / bg-deep / surface / ink / ink-2 / ink-soft / ink-mute / line /
  terra / sage / gold / navy` (у каждого `-2 / -soft / -tint` варианты)
- **Типографика:**
  - Display: `Bricolage Grotesque` (h1–h4, статистика, значки)
  - Body: `Manrope` (всё остальное)
  - Mono: `JetBrains Mono` (чипы длительности / XP)
- **Радиус:** 6 / 8 / 10 / 12 / 14 / 18 / 999
- **Отступы:** 4 / 8 / 10 / 12 / 14 / 16 / 18 / 20 / 24 / 28 / 32
- **Тени:** `shadow-sm / shadow-md / shadow-lg`

Если нужен цвет вне токен-сета — **остановись и спроси**. Скорее всего, уже есть
подходящий токен.

## Соглашения по коду

- **TypeScript strict.** Никакого `any` без явного `// @ts-expect-error` с пояснением.
- **Server components по умолчанию.** `'use client'` только при наличии state,
  обработчиков событий или browser API.
- **Server actions для мутаций.** Прогресс, ответы на квест, клейм бонуса —
  через actions, не через `/api/*` (исключение: вебхуки).
- **Нет client-side fetching для начальных данных.** Supabase читается в server
  component, пропы передаются вниз.
- **Colocation.** Компоненты, хуки и типы фичи живут в одной папке. `components/ui/`
  только для shadcn-примитивов.
- **Именование:**
  - Файлы: `kebab-case.tsx`
  - Компоненты: `PascalCase`
  - Хуки: `useThing`
  - Server actions: глагол первым (`markLessonDone`, `claimBonus`, `checkDailyQuest`)
- **Никаких barrel-файлов** (`index.ts` с re-exports).

## Supabase — соглашения

- У каждой пользовательской таблицы **RLS включён**. Политика — в том же файле миграции,
  что и таблица.
- FK используют `on delete cascade`, если дочерняя запись должна умирать с родителем.
- Схема: `supabase/migrations/*.sql`. **Никогда не редактируй существующие миграции** —
  только новые.
- После каждой миграции: `supabase gen types typescript --local > lib/supabase/types.ts`
  и коммит результата.
- Аутентификация: `getUser()`, а не `getSession()`. Service role key — только на сервере.
- Квесты (`answers`, `progress`) проверяются server-side. `correct`-флаги **никогда**
  не должны попасть в клиентский бандл. Используй `import 'server-only'`.

## Контент

- Весь текст — **на русском языке**.
- Тон: профессиональный + тёплый, обращение на **Вы** (в объяснениях квестов).
- Любопытные факты: одно предложение, фактологично, по возможности со ссылкой на источник.
- Упоминания L1-курса — **только в уроках 2 и 5**. Не рассыпай по всему курсу.
- `src/lib/course-data.ts` и файлы в `content/` — источник истины. Не хардкодь строки
  в компонентах.

---

## Безопасность ответов квеста

- Банк вопросов: `src/lib/` в `.ts` с именованными экспортами и `import 'server-only'`.
- Server action проверяет ответ, клиенту возвращается только `{ correct: boolean, explanation: string }`.
- Правильные варианты (`correct: true`) **никогда** не сериализуются в JSON для клиента.

---

## Доступность

- Все интерактивные элементы фокусируемы и имеют видимый focus ring (Tailwind default — ок,
  не убирай).
- Контраст: минимум AA. `terra` на светлом фоне при малых размерах — пограничный случай,
  предпочитай `ink` на `bg` для основного текста.
- Модальное окно урока: trap focus, восстановление при закрытии, ESC закрывает.
- Маркеры на роадмапе: `aria-label` с заголовком урока и состоянием, например
  `"Урок 3, в процессе"`.

## Definition of Done

Фича считается готовой, когда:

1. Соответствует дизайн-токенам и отступам из хендоффа.
2. Работает на 375 / 768 / 1280 px без горизонтального скролла.
3. Переживает офлайн-перезагрузку (нет падений из-за шрифтов / иконок).
4. Есть нужные состояния: loading / empty / error.
5. Server actions возвращают типизированные ошибки, UI их показывает.
6. Нет console errors и React warnings в dev.

Если что-то не выполнено — **напиши об этом явно**. Не делай вид, что готово.
